# Glossário HealtFlow

Este glossário contém definições de termos técnicos, médicos e de negócio utilizados no sistema HealtFlow.

---

## A

### API (Application Programming Interface)
Interface de programação que permite a comunicação entre diferentes sistemas de software.

### Agendamento
Processo de marcação de consultas, exames ou procedimentos médicos.

### Anamnese
Entrevista realizada pelo profissional de saúde para coletar informações sobre o histórico de saúde do paciente.

### Autenticação
Processo de verificação da identidade de um usuário no sistema.

### Autorização
Processo de determinar quais recursos um usuário autenticado pode acessar.

### Audit Log
Registro de todas as ações realizadas no sistema para fins de rastreabilidade e compliance.

---

## B

### Backend
Parte do sistema responsável pelo processamento de dados, lógica de negócios e comunicação com o banco de dados.

### Badge
Conquista visual que o paciente pode receber por cumprir metas de saúde no sistema de gamificação.

### Boleto
Forma de pagamento brasileira via documento bancário.

### Bull
Biblioteca para gerenciamento de filas de tarefas assíncronas em Node.js.

---

## C

### CFM (Conselho Federal de Medicina)
Órgão normativo e fiscalizador do exercício da medicina no Brasil.

### CID (Classificação Internacional de Doenças)
Sistema de classificação de doenças e outros problemas de saúde publicado pela OMS.

### CNES (Cadastro Nacional de Estabelecimentos de Saúde)
Sistema de informações sobre estabelecimentos de saúde no Brasil.

### CNS (Cartão Nacional de Saúde)
Número de identificação do usuário do SUS.

### CORS (Cross-Origin Resource Sharing)
Mecanismo de segurança que permite ou restringe solicitações de recursos entre diferentes domínios.

### CPF (Cadastro de Pessoa Física)
Documento de identificação de pessoa física no Brasil.

### CRM (Conselho Regional de Medicina)
Número de registro profissional do médico em seu estado.

### CRUD
Acrônimo para Create, Read, Update, Delete - operações básicas de manipulação de dados.

---

## D

### Dashboard
Painel visual que apresenta informações e métricas importantes do sistema.

### DATASUS
Departamento de Informática do SUS, responsável pelos sistemas de informação em saúde.

### DDD (Domain-Driven Design)
Abordagem de design de software focada no domínio de negócio.

### DevOps
Conjunto de práticas que integram desenvolvimento de software e operações de TI.

### Docker
Plataforma para desenvolvimento, envio e execução de aplicações em containers.

### DTO (Data Transfer Object)
Objeto usado para transferir dados entre subsistemas de uma aplicação.

---

## E

### E2E (End-to-End)
Tipo de teste que verifica o fluxo completo de uma funcionalidade.

### Endpoint
URL específica de uma API que recebe e responde requisições.

### Expo
Plataforma e conjunto de ferramentas para desenvolvimento de aplicações React Native.

---

## F

### FHIR (Fast Healthcare Interoperability Resources)
Padrão internacional para troca de informações de saúde.

### Firebase
Plataforma do Google para desenvolvimento de aplicativos móveis e web.

### Frontend
Parte do sistema responsável pela interface do usuário.

---

## G

### Gamificação
Uso de elementos de jogos em contextos não relacionados a jogos para engajar usuários.

### Guard
Componente de segurança que controla o acesso a rotas ou recursos.

### Git
Sistema de controle de versão distribuído.

### GraphQL
Linguagem de consulta para APIs (não utilizada no HealtFlow, que usa REST).

---

## H

### Helmet.js
Middleware que ajuda a proteger aplicações Express/NestJS configurando headers HTTP de segurança.

### HL7 (Health Level Seven)
Organização que desenvolve padrões para troca de dados de saúde.

### HPA (Horizontal Pod Autoscaler)
Recurso do Kubernetes que ajusta automaticamente o número de pods baseado em métricas.

### HTTPS
Protocolo de transferência segura de hipertexto.

---

## I

### ICP-Brasil
Infraestrutura de Chaves Públicas Brasileira, sistema que garante autenticidade e validade jurídica de documentos eletrônicos.

### Interceptor
Componente que intercepta requisições/respostas para processamento adicional.

### IoC (Inversion of Control)
Princípio de design onde o controle de objetos é transferido para um container.

---

## J

### JWT (JSON Web Token)
Padrão para autenticação e troca de informações entre partes de forma segura.

### JSON (JavaScript Object Notation)
Formato leve de troca de dados.

---

## K

### Kubernetes (K8s)
Plataforma de código aberto para automação de implantação, escalonamento e gerenciamento de aplicações em containers.

---

## L

### LGPD (Lei Geral de Proteção de Dados)
Lei brasileira que regulamenta o tratamento de dados pessoais.

### LiveKit
Plataforma de comunicação em tempo real para vídeo e áudio.

### LOINC (Logical Observation Identifiers Names and Codes)
Sistema de identificação universal para observações clínicas e laboratoriais.

---

## M

### Middleware
Software que atua como ponte entre diferentes aplicações ou componentes.

### MinIO
Sistema de armazenamento de objetos compatível com S3.

### Monorepo
Estratégia de desenvolvimento onde múltiplos projetos residem em um único repositório.

### MVC (Model-View-Controller)
Padrão arquitetural que separa a aplicação em três componentes interconectados.

---

## N

### NestJS
Framework Node.js para construção de aplicações server-side eficientes e escaláveis.

### Next.js
Framework React para produção com renderização do lado do servidor.

### Nginx
Servidor web de alto desempenho e proxy reverso.

### Node.js
Ambiente de execução JavaScript server-side.

---

## O

### OAuth
Protocolo padrão de autorização.

### ORM (Object-Relational Mapping)
Técnica que mapeia objetos para tabelas de banco de dados.

### OTA (Over-The-Air)
Atualizações enviadas remotamente sem necessidade de nova instalação.

---

## P

### PIX
Sistema de pagamento instantâneo brasileiro.

### PostgreSQL
Sistema de gerenciamento de banco de dados relacional open source.

### Prescrição Digital
Receita médica emitida e assinada eletronicamente.

### Prisma
ORM moderno para Node.js e TypeScript.

### Prontuário Eletrônico
Registro digital das informações de saúde do paciente.

---

## Q

### Query
Consulta a um banco de dados ou API.

### Queue (Fila)
Estrutura de dados que processa itens na ordem em que foram adicionados.

---

## R

### RBAC (Role-Based Access Control)
Modelo de controle de acesso baseado em papéis/funções.

### React
Biblioteca JavaScript para construção de interfaces de usuário.

### React Native
Framework para desenvolvimento de aplicativos móveis usando React.

### Redis
Sistema de armazenamento de estrutura de dados em memória.

### REST (Representational State Transfer)
Estilo arquitetural para desenvolvimento de serviços web.

### RNDS (Rede Nacional de Dados em Saúde)
Plataforma nacional de interoperabilidade de dados de saúde do SUS.

---

## S

### S3 (Simple Storage Service)
Serviço de armazenamento de objetos da Amazon Web Services.

### SBIS (Sociedade Brasileira de Informática em Saúde)
Entidade que promove padrões de informática em saúde no Brasil.

### SendGrid
Plataforma de entrega de email transacional.

### SOAP (Subjective, Objective, Assessment, Plan)
Método de documentação usado por profissionais de saúde para registrar consultas.

### Stripe
Plataforma de processamento de pagamentos online.

### SUS (Sistema Único de Saúde)
Sistema público de saúde brasileiro.

### Swagger
Conjunto de ferramentas para design, construção e documentação de APIs REST.

---

## T

### Tailwind CSS
Framework CSS utilitário para estilização de interfaces.

### Telemedicina
Prestação de serviços médicos à distância utilizando tecnologias de comunicação.

### TISS (Troca de Informação em Saúde Suplementar)
Padrão para troca eletrônica de dados entre operadoras de planos de saúde e prestadores.

### TLS (Transport Layer Security)
Protocolo de segurança para comunicações na internet.

### Token
Sequência de caracteres usada para autenticação ou autorização.

### TOTP (Time-based One-Time Password)
Algoritmo para geração de senhas descartáveis baseadas em tempo.

### Turborepo
Ferramenta de build de alta performance para monorepos JavaScript/TypeScript.

### Twilio
Plataforma de comunicação em nuvem para SMS, voz e mensagens.

### TypeScript
Superconjunto tipado do JavaScript.

---

## U

### UI (User Interface)
Interface visual com a qual o usuário interage.

### UUID (Universally Unique Identifier)
Identificador único universal de 128 bits.

### UX (User Experience)
Experiência geral do usuário ao interagir com um produto ou serviço.

---

## V

### Validação
Processo de verificar se dados atendem a critérios específicos.

### Vital Signs (Sinais Vitais)
Medidas de funções corporais básicas: pressão arterial, frequência cardíaca, temperatura, etc.

---

## W

### Webhook
Callback HTTP que permite notificações em tempo real entre sistemas.

### WebSocket
Protocolo de comunicação bidirecional em tempo real.

---

## Z

### Zod
Biblioteca TypeScript para validação de esquemas com inferência de tipos.

---
